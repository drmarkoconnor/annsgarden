# Ann's Garden Planner Handoff - 2026-05-14

## Current State

Ann's Garden Planner is a mobile-first Next.js app hosted on Netlify from GitHub, backed by Supabase for database, auth and private photo storage. It is now past the static prototype stage and is using the live Supabase database.

The app is intentionally still compact and personal-project focused. It uses server-rendered App Router pages, server actions for mutations, and small contextual forms rather than a large admin interface.

## Working Rules

Read all files in `/docs` before making broad changes. The docs are the source of truth for product intent, scope control and build phases.

Keep changes mobile-first and practical. Do not add large future-facing systems unless the user explicitly asks. The project has benefited from small, deployable slices.

Commit after each working phase, run validation, and push to `main` when the user wants Netlify to build.

## Data Access Pattern

Server data loaders live under:

```text
src/lib/*/data.ts
```

Server actions live under:

```text
src/lib/*/actions.ts
```

Most mutations use `createSupabaseAdminClient()` with `SUPABASE_SECRET_KEY`, but are protected by `requireSignedIn()` first. RLS is enabled on tables, but the app currently relies on server-side auth gates and the admin client for private personal use.

Contextual forms use:

```text
src/lib/navigation/return-path.ts
```

This keeps users on the same area/page after adding tasks, photos, diary notes or plants.

## Important Gotchas

The seeded profiles were removed. Auth users still exist in Supabase Auth, but `public.profiles` is separate. If "By" dropdowns are empty, add real profile rows in More.

The seeded categories were removed. Task category dropdowns may be empty until real categories are added in More. This is okay; categories are optional in most create flows.

The seeded pest issues were removed. Pest watch may look empty until real issues are added or a future curated starter list is intentionally reintroduced.

The app still assumes one garden through `ANN_GARDEN_ID`. Multi-garden support is not implemented.

Forgot password is not implemented yet. Supabase Auth supports it, but the UI does not expose it.

Camera access is browser-mediated. The app uses file inputs with `accept="image/*"`; on phones this usually offers camera/photo library, but there is no native camera integration.

AI plant ID is intentionally a suggestion workflow. It should not automatically overwrite plants.

## Current Clean Data Expectations

After the cleanup migration, a newly opened app should look sparse:

```text
Garden: likely no areas/plants except user-created rows
Tasks: likely no tasks except user-created rows
Diary: likely no notes except user-created rows
Photos: user-uploaded photos should remain
More: profiles/categories may need real rows
Plant Assist: prior AI suggestions should remain unless they referenced deleted seed data
```

If user-uploaded photos had been linked to seeded areas/plants, those links may now be null. The photo records themselves should remain.
